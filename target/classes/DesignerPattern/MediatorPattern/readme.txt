中介者模式：

    定义： 用一个中介者对象取封装一些列的对象交互，中介者使对象不需要显示的相互作用，
          从而使其耦合松散，而且可以独立地改变它们之间的交互

    角色：
          1、Mediator 抽象中介者 : 用于各个角色之间的通信
          2、ConcreteMediator 具体中介者：具体中介者角色通过协调同事角色实现协作行为，因此它必须依赖各个同时角色
          3、Colleague 同事角色：每个同事角色都知道中介角色，而且与其他的同事角色通信必须通过中介者协作，行为分为：自发行为和依赖行为

    优点：
          1、减少类间依赖，把原有的一对多依赖转换为一对一依赖

    缺点：
          1、中介者会膨胀得很大，逻辑复杂，同事类元多，中介者的逻辑越复杂

    使用场景：多个对象之间紧密耦合的情况：类图中出现蜘蛛网状结构
          1、N个对象之间产生了相互依赖关系
          2、多个对象有依赖关系，但是依赖的行为尚不确定或有可能发生改变的可能，降低变更引起的风险扩散
          3、产品开发。像mvc框架


    具体应用：
         1、MVC框架，C层把MV层耦合降低了
         2、机场调度
         3、媒体网关
         4、中介服务

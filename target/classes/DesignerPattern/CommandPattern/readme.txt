命令者模式

    定义： 命令模式是一个高内聚的设计模式，把请求方和执行方分开了
          将一个请求封装成一个对象，从而让你使用不同的请求把客户端参数化，
          对请求排队或者记录请求日志，可以提供命令的撤销和恢复功能

    角色：
         1、Receive：接受者角色，也就是干活的角色，命令传递到这里以后需要被执行
         2、Command：命令角色，需要执行的命令都在这里声明
         3、Invoker：调用者角色，接收命令，并且执行命令

    优点：
        1、类间解藕：调用者和执行者没有任何依赖关系，不需要了解到底是哪个执行者执行的
        2、可扩展性：Command的子类可以很方便扩展
        3、命令模式结合其他模式会更棒：结合责任链模式，实现命令族解析任务;结合模板方法模式，可以减少Command子类的膨胀问题

    缺点：
        1、N个命令就有N的Command的实现类，子类膨胀问题

    使用场景：
        1、有命令的地方就可以使用了

    扩展：封闭Receiver方式

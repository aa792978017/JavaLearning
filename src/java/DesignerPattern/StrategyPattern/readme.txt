策略模式：

    定义： 定义一组算法，将每个算法都封装起来，并且使它们可以相互替换

    优点：
        1、算法可以自由切换，只需要在容器更换不同策略即可
        2、避免使用多重条件判断
        3、扩展性好：只需要实现接口，重写方法即可

    缺点：
        1、策略类增多，一个策略一个类，复用可能性小
        2、所有策略需要对上层暴露（使用工厂模式等避免）

    场景：
        1、多个类只有在算法上少有不同的场景
        2、算法需要自由切换的场景
        3、需要屏蔽算法规则的场景

    注意事项：
        1、具体策略数量超过四个，需要考虑混合模式来解决策略膨胀等问题和对外暴露等问题

     扩展：
        策略枚举，个人感觉使用场景有限，不做过多了解
